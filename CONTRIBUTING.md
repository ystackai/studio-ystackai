# Contributing to Crew Factory

Thank you for your interest in contributing to the Crew Factory project! This document outlines how to contribute to the project.

## Project Overview

Crew Factory is a browser-based game where players can create, customize, and deploy teams of AI agents (crews) to complete missions and challenges. The project is built using web technologies and follows a modular, scalable architecture.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please:
1. Check if the issue already exists in our issue tracker
2. Create a new issue with a clear description
3. Include steps to reproduce (for bugs)
4. Add appropriate labels (bug, enhancement, etc.)

### Making Changes

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes following the project's coding standards
4. Write tests for your changes where applicable
5. Submit a pull request with a clear description

### Code Style

- Follow the existing code style in the repository
- Use meaningful variable and function names
- Write clear, concise comments
- Ensure code is well-documented
- Follow the project's architectural patterns

### Testing

- Write unit tests for new features
- Ensure existing tests still pass
- Test across different browsers if relevant
- Document test procedures in the pull request

### Documentation

- Update documentation when making changes
- Keep API documentation up-to-date
- Add code comments where necessary
- Update the README if needed

## Development Process

1. Create an issue describing your proposed changes
2. Discuss the implementation approach with the team
3. Create a branch and make your changes
4. Submit a pull request
5. Address any feedback
6. Get your changes merged

## Code of Conduct

All contributors are expected to follow our Code of Conduct, which promotes a welcoming and inclusive environment for everyone.

## Questions?

If you have any questions about contributing, please reach out to the project maintainers.
